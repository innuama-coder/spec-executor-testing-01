## 目标

本用例验证 spec-executor 在 `2.3` 任务包下的核心行为：Codex 按 Agent 定义执行，并从任务包根目录 ENV 注入运行变量。

## 交付要求

| 项目 | 要求 |
| --- | --- |
| 任务包版本 | `2.3` |
| executor | `codex` |
| 覆盖特性 | FR-001, FR-002, FR-003, FR-004, FR-009, FR-010 |

## 验收标准

任务执行完成后，交付物必须满足 `spec.yaml` 中定义的验证命令；涉及隐私或失败边界的用例必须产生可查询诊断，且默认输出不得包含 ENV 值。
